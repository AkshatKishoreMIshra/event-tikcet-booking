BUSINESS RULE ENFORCEMENT

1. Booking allowed only if event_status = upcoming
2. Ticket becomes invalid after event_date
3. Refund allowed only before event_date
4. If refund approved:
   - order_status = refunded
   - seat_status = available
   - ticket_status = cancelled
5. Ticket becomes invalid once marked as used
6. Users cannot exceed max_tickets_per_user
7. Payment required before ticket generation
8. Seat cannot be double-booked
